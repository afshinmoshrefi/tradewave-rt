# TradeWave Realtime - Architecture Map

> Generated 2026-06-15 by a multi-agent firsthand deep-read of the codebase (11 subsystem readers + synthesis + risk critic, 13 agents, ~1.17M tokens, 265 source reads). This is the durable reference; the launch punch-list lives in `LAUNCH_RISKS.md`.

## System summary

TradeWave Realtime is a Flask monolith (single SQLite DB in WAL mode, served by gunicorn -w 2 --threads 8 behind a Cloudflare tunnel, User=root) that delivers Anne-Marie Baiynd's intraday trading method as a paid daily-levels product plus an educational AI "coach" trained on her, with a partner workspace for her to teach/correct it and an operator room for site operations.

BOOT (app/__init__.py create_app, run once per gunicorn worker -> twice, concurrently): load Config (secrets.env injected by systemd, else app/.env; SECRET_KEY auto-generates a throwaway if unset), wrap wsgi_app in ProxyFix(x_for/x_proto/x_host=1) so url_for builds https, db.init_app, register 7 blueprints (auth, main, chat, admin, ingest, mentor, billing), register the `md` Jinja filter + the inject_globals context processor (BRAND/PARTNER/COACH_NAME/COACH_LABEL + persona.AI_DISCLOSURE/RISK_DISCLAIMER + LLM/WORKOS/BILLING feature flags + current_user, injected into EVERY template), register a before_request that resets g._cached_user, then under app_context run db.create_all() -> _run_micro_migrations() (idempotent hand-maintained ALTER TABLE ADD COLUMN dict, SQLite-only) -> rag.rebuild_index() (wrapped in a bare except that only warns). There is no Alembic; the micro-migration dict must be kept in lockstep with models.py and cannot add the model's UniqueConstraints/Indexes to an already-seeded DB. extensions.py sets NO busy_timeout/journal_mode/pool tuning, so concurrent boot ALTERs and up-to-16 concurrent request threads hit one un-hardened SQLite file.

MEMBER REQUEST LIFECYCLE: before_request nulls g._cached_user; security.current_user() lazily loads User by session['user_id'] and memoizes on g. Access is layered by decorators: login_required (identity), disclaimer_required (one-time accepted_ai_disclaimer ack, else redirect to /welcome) gates the coach/intake/profile/check-in, admin_required = is_staff (admin OR partner) gates Anne-Marie's whole workspace, operator_required = is_admin only gates the Operations room and role assignment. A logged-in member hitting /app (dashboard, app/main.py:24) is redirected into the 6-step server-driven intake until profile.intake_done; otherwise Today is assembled from latest_maps() (per-instrument DayMap cards with deterministic direction_sentences), today's insight Post, library/lesson counts, today's check-ins + checkin_phase (plan before 16:00 ET, review after), the next published+future Appearance, and a session-state "time_note". The same DayMap rows feed the coach and the NinjaTrader indicator feed, so Today, coach, and indicator share one source of truth.

LEVELS ENGINE (the deterministic spine; app/marketdata.py + app/levels.py + app/ingest.py): two write paths populate MarketCandle. (1) PUSH: the external keyprovider quote service POSTs a completed window to /api/ingest/candle, which requires a matching X-Ingest-Token (fails CLOSED, 401, if the token is unset/mismatched), allow-lists instrument+window, rejects implausible session_date (weekends, outside -5..+4 days), non-finite/non-positive prices, inconsistent OHLC, and negative volume, then idempotently upserts a body-only source='capture' candle (refusing to overwrite eodhd/reconciled rows) and immediately rebuilds that instrument's DayMap. (2) PULL: the 15-min levels-sync systemd heartbeat GETs /levels_rt for capture snapshots at exact ET boundaries and hourly EODHD intraday history (IntradayBar). Overnight, reconcile_candle overwrites a capture row with EODHD truth, preserving capture_open/capture_close and recording recon_drift = max(|open delta|,|close delta|) with a 0.5pt tolerance, promoting source to 'reconciled' (history truth always wins; drift>0.5 is logged-only, never alerted/surfaced). build_map then derives the master candles, runs the stair-step (strict body no-overlap), in-charge box, verdict (up/down/sideways/pending with clean-break resolution), MOC volume gate (3:30 vs 3:00, FULL 1.20x / GRAY 0.80x), and the 30-min 50/200 SMA stack+slope gate (needs >=206 bars, else degrades to "building"), and persists DayMap.payload + status (ok/partial/pending). today_context()/now_context() render the coach's NOW block (live ET clock + session phase + published-Appearances-only schedule) and TODAY block (the published map numbers, levels marked '~' when still capture-approximate). The LLM never writes levels or the direction card - direction_sentences uses fixed educational templates.

THE COACH (app/persona.py + app/llm.py + app/rag.py + app/chat.py + app/mentor.py): persona.py is the single source of the coach identity, voice, and ALL compliance hard rules (educational-only, never individualized buy/sell, never pick a size/contract count or work back from account/dollar-risk, never an income number/range/%/timeline, no em dashes, AI-not-Anne-Marie, never promise her schedule). The decided architecture (llm.py docstring) gives the coach the ENTIRE published+approved method corpus (method_corpus, filtered published=True AND status='approved', non-her_words entries tagged "hedge if asked") as a prompt-cached, byte-stable system prefix shared by every member; the volatile block (now_context + today_context + mentor.member_context + extra_instruction) renders AFTER the cache breakpoint so per-member data never enters the shared cache. RAG/TF-IDF retrieval survives ONLY to pick UI citation chips and to power the demo-mode fallback when ANTHROPIC_TOKEN is unset (note rag.rebuild_index filters published=True ONLY, NOT status, so a published+draft entry is excluded from the answer body but still retrievable as a chip/demo quote). A chat turn (chat.stream_message SSE, or the non-streaming twin): enforce DAILY_CHAT_CAP=60, build member_context (teaching-only, "never mention account sizes", '' for the free Discord tier), stream stream_reply deltas (each em-dash-stripped), then run correct_level_typos (deterministic digit-flip repair against the real map levels), persist the assistant ChatMessage (rating=-2 if the reply contains the literal "flagged this for Anne-Marie"), and post-stream run maybe_update_summary (Haiku rolls a bounded, money-blind coaching_summary every 8 messages). Two-model split (config.py): CHAT_MODEL=Sonnet for member-facing coach/intake/check-in/briefing; CLAUDE_MODEL=Haiku for the summary, lesson drafting, the privacy sanitizer, and the eval judge. The ONLY programmatic backstops on a live reply are _no_emdash and correct_level_typos; the individualized-advice/sizing/income rules are prompt-only (the advice-seeking detector influences only the demo fallback).

ADMIN / CORRECTION LOOP (app/admin.py): the front door routes the operator (is_admin) to the Operations room and the partner (is_staff, not is_admin) to her workspace. The teaching loop: a member thumbs-down (rating=-1 via rate_message) or a coach deferral (rating=-2) lands a ChatMessage in the /reviews queue; Anne-Marie opens it (the prior member question is lazily anonymized by privacy.sanitize_for_admin and cached on ChatMessage.sanitized - she never sees raw member text), and /teach drafts a lesson via Haiku (generalizing day-specific prices/dates), which teach_publish writes as a KnowledgeEntry with provenance='her_words'/status='approved'/published=True, flips the flag reviewed=True, and calls rag.rebuild_index(). Generic _save_knowledge validates category/provenance but ALWAYS forces status='approved' (the design's "AI/team drafts wait as draft" hold has no implemented path). Every knowledge mutation rebuilds the index AND invalidates the method_corpus cache (keyed on count+max(updated_at)) - but rebuild_index runs only in the worker that served the request, so with -w 2 the other worker serves a stale corpus until restart. The daily insight Post and schedule note are free-text, published instantly to members and into the coach's today-context with no scrub/review. The Operations room shows aggregate KPIs, a candle drift table (recon_drift>0.5), and a bounded People & roles panel; ops_set_role is the only role-grant mechanism (roles live in this app's DB, never WorkOS) and refuses to change the acting operator's own role (the sole lockout guard).

BILLING + ROLES (app/billing.py): RT products live on a Stripe account SHARED with TradeWave EOD, tagged product_line=rt with rt_* lookup keys so Net RT Revenue is computable for Anne-Marie's 35% monthly statement. checkout()/portal() are session-gated; the signed webhook verifies the Stripe signature over the RAW body, filters to RT via _is_rt_subscription (metadata/lookup-key, so EOD purchases never mint an RT row), resolves the user by metadata.rt_user_id then stripe_customer_id, and upserts the local Subscription mirror (status written verbatim from Stripe). active_subscription() (status in ACTIVE_STATUSES = {active,trialing,past_due}) is the single entitlement signal. ROLE ASSIGNMENT: on WorkOS first login or dev signup, role = admin if email in ADMIN_EMAILS, partner if in PARTNER_EMAILS, else member; WorkOS asserts only identity (email + workos_user_id, merging local accounts by email). Identity is intended to federate with EOD via shared WorkOS AuthKit; V1 ships session email/password as the live path with WorkOS config-gated. CRITICAL GAP: BILLING_REQUIRED is honored in exactly one place (Discord role-sync); the member web routes (/app, /app/feed, /app/library, /app/account, the coach) are gated by @login_required only, so flipping BILLING_REQUIRED at launch does NOT paywall the website - any free logged-in account keeps full access to the levels, library, and coach.

PROACTIVE COMMS + INGESTION (app/notify.py + app/briefing.py + app/discord_bot.py): notify is Resend-only; transactional kinds (welcome) bypass caps, coach kinds obey session-days-only + quiet hours (20:00-06:30 ET) + max one per member per 24h, every send ledgered to NotificationLog, every footer carrying the AI+risk disclosure (only send_welcome is wired today). The weekday briefing timer pulls Anne-Marie's morning YouTube video (yt-dlp captions) -> Sonnet digest constrained to today's real DayMap numbers -> deterministic _scrub_numbers drops any sentence with an off-map 4-6 digit price -> Post(kind='briefing'), which surfaces in the feed and is injected into every coach's today_context labeled "an AI digest". The always-on Discord bot (separate systemd service, installed-but-disabled until tokens exist) runs two tiers: #ask-the-coach (include_today=True) and #coach-preview (evergreen only, include_today=False, FREE_DAILY_CAP=5/day in-memory) so today's map never crosses the paywall; it posts the morning map after 07:30 ET, mirrors insight posts, and hourly syncs Member/Free roles to subscription state.

OPS / QUALITY GATE (evals/run_evals.py + deploy/ + backup_db.py): the pre-launch eval battery exercises the real coach stack (create_app + generate_reply) across GUARDRAIL/INCOME/METHOD/VOICE/TODAY suites with a Haiku judge (majority-of-3 on FAIL only) plus regex tripwires, and exits 1 if ANY guardrail or income case fails. install.sh enables 4 always-on units (web app, levels.timer 15-min, backup.timer nightly 08:30 server-local, briefing.timer weekday ET) and leaves the bot + cloudflared tunnel installed-but-disabled; secrets.env, the venv, the tunnel, and DB restore are explicitly out-of-band. backup_db.py runs a nightly SQLite online .backup() with 14-day pruning, on the same host, with no integrity check or failure alert.

## Cross-subsystem data flows

- BOOT (per gunicorn worker, x2 concurrent): wsgi.py -> create_app() -> Config(secrets.env) -> ProxyFix -> register 7 blueprints + context processor + before_request(reset g._cached_user) -> db.create_all() -> _run_micro_migrations() (idempotent SQLite ADD COLUMN) -> rag.rebuild_index() (bare-except, warn-only).
- PER REQUEST AUTH: before_request nulls g._cached_user -> security.current_user() loads User by session['user_id'] and memoizes on g -> decorators gate by login_required / disclaimer_required (accepted_ai_disclaimer) / admin_required (is_staff) / operator_required (is_admin).
- LEVELS CAPTURE -> MAP (push): keyprovider POST /api/ingest/candle -> X-Ingest-Token check (fail-closed) -> instrument/window allow-list + session_date/price/OHLC/volume plausibility guards -> upsert MarketCandle source='capture' (skip if eodhd/reconciled) -> build_map() rebuilds that instrument's DayMap immediately.
- LEVELS CAPTURE -> MAP (pull/heartbeat): levels-sync 15-min timer -> fetch_keyprovider_levels (/levels_rt body-only captures) + hourly EODHD IntradayBar -> build_map().
- RECONCILE -> AUDIT: overnight EODHD truth -> reconcile_candle overwrites a capture row, preserves capture_open/close, records recon_drift=max(|dO|,|dC|) (0.5pt tolerance), promotes source='reconciled' -> drift>0.5 logged-only (no alert/DayMap flag).
- MAP BUILD: IntradayBar/MarketCandle -> stair_step (strict body no-overlap) -> in_charge box -> verdict (up/down/sideways/pending) -> MOC gate (3:30 vs 3:00 vol) -> SMA gate (30m 50/200, >=206 bars) -> DayMap.payload + status (ok/partial/pending).
- MAP -> COACH GROUNDING: latest_maps() -> today_context()/now_context() -> llm._build_chat_request volatile block (after cache breakpoint) -> coach reply -> correct_level_typos (digit-flip repair vs real levels) -> member; today_context gated behind include_today (paid), now_context to every tier.
- COACH PROMPT CACHE: persona.SYSTEM_PROMPT + SIGNATURE_NOTES + method_corpus() (published+approved, byte-stable, shared cache prefix) | now_context + today_context + member_context + extra_instruction (volatile, after breakpoint) -> Anthropic Sonnet stream -> _no_emdash on every delta.
- KNOWLEDGE AUTHORING -> CORPUS: admin /teach or /knowledge writes KnowledgeEntry -> rag.rebuild_index() (published only) + method_corpus cache invalidates on next call (count+max(updated_at)) -> but only in the worker that served the admin request (-w 2 staleness).
- CORRECTION LOOP: member thumbs-down (rating=-1) or coach deferral (rating=-2, literal 'flagged this for Anne-Marie') -> ChatMessage in /reviews queue -> sanitize_for_admin (Haiku, cached on ChatMessage.sanitized) anonymizes the prior member question -> Anne-Marie /teach -> Haiku drafts lesson -> teach_publish writes provenance='her_words'/status='approved'/published=True + reviewed=True -> rebuild_index.

- STRIPE -> ENTITLEMENT: Stripe webhook (signature verified over raw body) -> _is_rt_subscription product-line filter (EOD purchases excluded) -> resolve user by rt_user_id then stripe_customer_id -> upsert Subscription mirror (status verbatim) -> active_subscription() (ACTIVE_STATUSES) gates Discord role + indicator feed (but NOT the member website).

## Global invariants (must never break)

- AI DISCLOSURE IS ALWAYS PRESENT AND TRUTHFUL: the coach is always presented as an AI trained on Anne-Marie, never as her. inject_globals unconditionally injects AI_DISCLOSURE/RISK_DISCLAIMER into every template; disclaimer_required gates the coach behind a one-time accepted_ai_disclaimer ack; persona.SYSTEM_PROMPT, the chat GREETING, every Discord reply (DISCLOSURE suffix), and every email footer restate it.
- EDUCATIONAL-ONLY, NEVER INDIVIDUALIZED: the coach must never give an individualized buy/sell call, pick a position size/contract count for a member, work backward from account/prop/dollar-risk, or give an income number/range/%/timeline. Enforced by persona.SYSTEM_PROMPT (rules 1 and 5), reinforced in member_context and today_context, and gated at eval time by the GUARDRAIL+INCOME suites (run_evals exits 1 on any failure). On the live path the only deterministic backstops are em-dash strip + level-typo repair; the advice/sizing/income rules are prompt-only.
- NO MEMBER MONEY, POSITIONS, OR BALANCES STORED: UserProfile has no P&L/balance/position columns and its docstring forbids them; scrub_sensitive masks money/size phrases on every member-derived text written to long-term memory (intake answers, profile edits, check-ins, the rolling summary prompt), deliberately preserving bare price levels like 7491.00.
- MEMBER TEXT IS ANONYMIZED BEFORE IT REACHES ANNE-MARIE: every partner/admin display of a member's words passes through sanitize_for_admin (Haiku, with scrub_sensitive as a never-raising deterministic fallback and output post-filter), cached on ChatMessage.sanitized; the only raw text on those surfaces is the coach's own answer, and members are identified only as 'Member {id}'.
- PROVENANCE / RELEASE GATE: only her_words/her_approved is pre-approved; AI/team content should be held as draft, and only status='approved' published entries reach the coach's answer body (method_corpus filters published AND status='approved'); team_inference entries that do reach it are tagged so the coach hedges authorship. (Known drift: _save_knowledge/teach_publish force status='approved' so the draft-hold path is unimplemented, and rag.rebuild_index filters published only, so a published+draft entry can still surface as a citation chip / demo quote.)
- LEVELS ARE DETERMINISTIC, IDENTICAL FOR ALL MEMBERS, AND NEVER INVENTED: the direction card and level numbers come 100% from fixed templates + the published DayMap (the LLM never authors them); today_context labels the map identical-for-all/educational-reference; correct_level_typos repairs digit-flipped levels; the briefing _scrub_numbers and the TODAY eval reject any price not on the real map. Approximate (capture-only) levels are always flagged with '~'.
- HISTORY-TRUTH PRECEDENCE & AUDITABILITY OF LEVELS: a 'capture' candle may never overwrite an 'eodhd'/'reconciled' row (enforced in _upsert_capture, the ingest endpoint, and extract_candle); reconciliation preserves capture_open/capture_close/recon_drift so a bad capture is diagnosable after EODHD truth overwrites it; DayMap UniqueConstraint(instrument, session_date) + a session cap keep one authoritative map per instrument/day.
- INGEST WRITE SURFACE IS TOKEN-GATED AND PLAUSIBILITY-FENCED: /api/ingest/candle requires a matching X-Ingest-Token and fails CLOSED if the token is unset; it rejects implausible session_date, non-finite/non-positive prices, inconsistent OHLC, and negative volume before any DB write.
- THE COACH CITES HER SCHEDULE ONLY FROM PUBLISHED APPEARANCES AND NEVER PROMISES TIMING: Appearance is the sole sanctioned schedule source; now_context emits only published+future rows and appends 'Outside this list, never promise when she will appear'; the Today next_live card filters published+future only.
- MONEY / ENTITLEMENT TRUTH COMES ONLY FROM STRIPE, RT-SCOPED: every RT product/price/customer/subscription carries product_line=rt and rt_* lookup keys (the basis for Anne-Marie's 35% Net RT Revenue statement); _is_rt_subscription keeps shared-account EOD purchases from minting RT rows; active_subscription() over ACTIVE_STATUSES is the single entitlement signal; the webhook verifies the Stripe signature over the RAW body and the founding-rate flag must never silently flip.
- PROACTIVE-SEND ACCOUNTABILITY: the coach voice never sells; every email is ledgered to NotificationLog; coach-voice sends are capped (session-days only, quiet hours 20:00-06:30 ET, max one per member per 24h); all email HTML is template-only (no LLM writes outbound email) with the AI+risk disclosure footer.
- ROLE / PRIVILEGE BOUNDARY OWNED BY THIS APP'S DB: WorkOS asserts identity only; roles come from ADMIN_EMAILS/PARTNER_EMAILS at create time and ops_set_role afterward; is_admin (operator) gates the Operations room, the member PII directory, and role changes, while is_staff (admin OR partner) gates Anne-Marie's workspace - so a partner can never reach member PII or change roles, and the operator cannot change his own role (the sole lockout guard).
- PER-USER MEMORY IS SILOED AND DELETABLE: member_context/coaching_summary/check-ins/chat history are all user_id-scoped (the free Discord tier passes member_context=''); ChatThread/ChatMessage and UserProfile cascade delete-orphan from User, and the delete-chats/delete-profile routes genuinely erase the conversation, the derived summary, and check-ins.
- NO EM DASHES ANYWHERE IN COACH/CUSTOMER OUTPUT: _no_emdash hard-strips em dashes on every coach return path (LLM, fallback, intake wrap-up, check-in, per-stream delta), the rolling summary and the briefing digest also strip them, and persona forbids them - a project-wide hard rule.
